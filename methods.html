Methods

Data Sources and Analytical Approach

All data are drawn from publicly available CMS administrative files. Analysis was conducted in Python using pandas; mapping in ArcGIS Pro and D3.js.

Data Sources

This analysis draws on three linked CMS data sources for each year from 2020 to 2025:

  • Part D Formulary Reference Files — drug-level coverage information including tier placement, prior authorization flags, step therapy requirements, and quantity limits for all plan-contract-drug combinations
  • Part D Plan Characteristics Files — contract type (MA-PD vs. PDP), service area, and plan-level enrollment identifiers
  • Part D Enrollment Files — county-level plan enrollment used to construct enrollment-weighted measures

Drug identification used RxNorm concept unique identifiers (RXCUIs) linked to four therapeutic classes: GLP-1 receptor agonists, SGLT-2 inhibitors, DPP-4 inhibitors, and insulin analogs.

Restrictiveness Scoring

A county-level formulary restrictiveness score (0–4 scale) was constructed by summing four binary indicators for each drug-plan combination:

Score = PA + ST + QL + High_Tier
PA = Prior Authorization required (0/1)
ST = Step Therapy required (0/1)
QL = Quantity Limit applied (0/1)
HT = High Tier placement (Tier 4+) (0/1)

County-level scores represent the enrollment-weighted average across all Part D plans operating in the county for each drug class and year. Counties with fewer than five plan entries were excluded from county-level analysis to ensure reliability.

Step 01
Drug Identification
RxNorm API (RxNav) used to identify all RXCUIs within four drug classes. RXCUI lists validated against CMS formulary file NDC crosswalk to ensure complete capture of brand, generic, and combination products.
Step 02
Formulary Analysis
For each year, formulary files merged with plan characteristics to compute drug-class-level PA rates, step therapy rates, quantity limit rates, and tier distributions by contract. National and state aggregates computed.
Step 03
County Aggregation
Plan-county linkage via CMS service area files. Enrollment-weighted county restrictiveness scores computed for each drug class and year. FIPS code standardization applied via SSA-to-FIPS crosswalk.
